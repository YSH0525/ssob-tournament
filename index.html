☁️ Supabase 연동

      

        총무 1명이 Supabase 무료 프로젝트를 만들고, 참가자들과 URL/anon key를 공유하면

        대회 데이터가 모든 기기에서 동기화됩니다. 실시간 반영 지원.
      


      

        📘 설정 가이드 (처음 1회)
        

          1) Supabase 프로젝트 생성

          → supabase.com 가입 → New project → 이름/비밀번호 설정


          2) 테이블 생성 (SQL Editor에서 아래 붙여넣기 → Run)

          
create table if not exists ssob_meets (
  id text primary key,
  data jsonb not null,
  updated_at timestamptz default now()
);
create table if not exists ssob_members (
  id text primary key,
  data jsonb not null,
  updated_at timestamptz default now()
);
alter table ssob_meets enable row level security;
alter table ssob_members enable row level security;
create policy "public read" on ssob_meets for select using (true);
create policy "public read" on ssob_members for select using (true);
create policy "public write" on ssob_meets for insert with check (true);
create policy "public write" on ssob_meets for update using (true);
create policy "public write" on ssob_members for insert with check (true);
create policy "public write" on ssob_members for update using (true);

          3) Realtime 활성화: Database → Replication → 두 테이블 체크

          4) URL/키 복사: Settings → API → Project URL + anon public key

          5) 참가자 공유: 아래 URL/키를 톡방에 공유 (anon 키는 공개 안전)
